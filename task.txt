1. Разработать REST API, которое должно включать в себя:
Подключание к БД,
Обработку исключаний и невалидных данных, возвращая ошибку с уникальным кодом(не рандомное число, а имеенно у каждой ошибки свой код),
Получение входных данных в виде JSON,
Возвращение данных в виде JSON,
Поддержка минимум 2 языков (Русский, English и т.д.), в возвращаемых ответах.

API должно включать в себя 3 метода:
Регистрация клиента,
Авторизация клиента,
Изменение данных клиента.

1.1 - регистрация клиента: Получить POST запрос с JSONом вида:"{"name": "White_Tomato", "login":"123123", "password":"123123", "language":"ru-RU"}". Добавить нового пользователя в БД и вернуть ответ в виде JSON: "{"result": 1}", Где '1'  - уникальный идентификатор пользователя, который был присвоен при регистрации.
В случае ошибки вернуть JSON следующего вида: "{"error": {"code": 1, "message": "Не удалось зарегистрировать пользователя: данный логин уже занят."} }".
'code' и 'message' для каждой ошибки свои.

1.2 Авторизация клиента. Получить POST запрос с JSONом: "{"login": "123123", "password": "123123", "language": "eu-US"}".
Найти пользователя, есть есть совпадение, логин и пароль верны, то вернуть JSON вида:"{"result": { "user": { "id": 1, "name": "White_Tomato", "date_registration": "2021-05-12 11:56:47"}}}"
То же самое с ошибками как и в '1.1'.

1.3 Обновление информации о пользователе. Получить JSON:"{"id": 1, "name": "Петька Петров Петькович", "language": "ru-RU"}". В БД обновить name на новое имя где идентификатор равен из JSON. Вернуть резутат или error.
